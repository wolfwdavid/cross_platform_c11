## Out of scope for this PR

- Option E (sidebar tab grouping by worktree) — ROADMAP.
- Worktree chips on non-terminal surfaces (markdown/browser) — S6 deferred.
- FSEvents-based cache invalidation (E2) — modest design upgrade, defer to follow-up if shipping `mtime`-based invalidation proves insufficient in practice.
- A second deriver (host/SSH target, container, kubectl, AWS profile) — E3 deferred; the abstraction is ready when needed.
- A pre-merge tagged-build smoke gate via computer-use (S10) — depends on `c11-computer-use` skill infrastructure that doesn't exist yet.
- A formal perf budget AC (S7) — log/counter instrumentation may be added by the delegator at its discretion but not required by ACs.

## Notes for the Result Validator

- Walk **only pre-merge-static rows** against the open PR. AC31–AC33 (post-merge-smoke) go to the "Operator smoke-pass checklist" section of the Validation Report.
- Highest-value audit rows: AC12 (cache against resolved HEAD), AC13 (stale-async), AC22–AC23 (hot-path discipline), AC24 (no duplicate render path), AC26 (settings key preserved).
- The validator does NOT re-run the unit tests themselves (CI already does that); the audit is about *whether each AC row's shape matches a real test in the PR*, not pass/fail re-execution.
- Cross-reference AC numbers in the delegator's commit messages or PR body if the delegator uses that convention — saves the validator from having to hunt.
